# Manual

## What is this
This manual provides a high-level overview of cliboa's concept, structure, and how to organize your projects for efficient ETL/ELT pipeline management.

## What is cliboa?
**cliboa** is a Python-based framework designed to easily implement sequential processes, primarily for ETL (Extract, Transform, Load) or ELT purposes.
By simply defining a scenario file (YAML) and specifying it as a parameter, you can flexibly combine and execute various ETL components.

While built with ETL in mind, cliboa is essentially a **workflow (pipeline) engine**. Its default modules are naturally geared toward data processing, but its high extensibility allows it to be used for a wide range of automation and processing tasks by implementing custom modules.

### What is project
In cliboa, a **Project** is the fundamental unit for scenario management.
A project serves the purpose of organizing all files related to a specific scenario under a single umbrella, keeping project-specific logic and configurations isolated from others.

## Directory Structure (Sample)
The following is a sample directory structure generated by cliboadmin. While this structure is recommended for a smooth start, it is provided as a sample, not necessarily a "best practice."

cliboa v3 is designed with flexibility in mind, and future updates may refine the recommended layout.

```
cb_app/
├── cliboa_run.py           # Main execution script
├── cliboa_environment.py   # Environment-specific configurations
└── scenario/
    └── additional_custom_module.py # Your custom scenario logic files
project/
└── sample_project/     # Project unit
    ├── scenario.yaml   # Scenario definition
    └── scenario/
        └── logic.py    # Project's specific logic
└── common/             # Common across projects
    └── scenario.yml
conf/
└── logging.conf
```

## Entrypoint of cliboa
There are two ways to use cliboa, depending on your needs:

1. **Standard Runner (QuickStart style)**
Use the `cliboa_run.py` script generated by `cliboadmin`. This is the easiest way to get started and is suitable for most ETL tasks where you want to execute scenarios via CLI.

2. **Library Integration (Customizable style)**
For users who need higher flexibility, cliboa can be imported as a library within your own Python scripts. This allows you to trigger cliboa pipelines as part of a larger application or integrate them into existing systems.
If you need to use this way, please create ScenarioManager's instance and call the execute method.
You can learn more about how to use ScenarioManager by reading the [ScenarioManager](/cliboa/core/manager.py) public class directly.

## How to setup cliboa_environment.py
In cliboa, environment values for cliboa are managed in `cliboa_environment.py`. 

To find out what to write, read the code comments in cliboa_environment.py(placed when you run QuickStart) or [default_environment.py](/cliboa/conf/default_environment.py).
